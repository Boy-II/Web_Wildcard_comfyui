# ComfyUI Wildcard 目錄監視功能

## 📋 功能概述

ComfyUI Wildcard 目錄監視功能允許您實時查看 `E:\Comfy_Wildcard` 目錄的結構，統計檔案和行數，並可選擇將其同步到資料庫中進行管理。

**新增日期**: 2025-12-07

---

## 🎯 主要功能

### 1. 目錄結構視覺化
- 樹狀顯示完整的目錄結構
- 顯示每個目錄包含的檔案數量和總行數
- 可摺疊/展開的目錄樹
- 顯示每個 TXT 檔案的詳細資訊（行數、大小、修改日期）

### 2. 統計資訊
- **TXT 檔案總數**: 顯示目錄中所有 .txt 檔案數量
- **總行數**: 統計所有 TXT 檔案的總行數
- **目錄數**: 顯示子目錄數量
- **最後掃描時間**: 記錄最近一次掃描的時間

### 3. 同步到資料庫
- 一鍵將 ComfyUI Wildcard 檔案同步到資料庫
- 自動根據目錄結構分類
- 檢測並跳過重複項目
- 顯示同步結果（成功、跳過、錯誤）

## 🚀 快速開始

### 訪問頁面
```
http://localhost:9000/comfy-monitor
```

或點擊導航欄的 **「ComfyUI 監視」**

### 使用步驟

1. **掃描目錄**
   - 點擊右上角「重新掃描」按鈕
   - 系統會掃描 `E:\Comfy_Wildcard` 目錄
   - 顯示完整的目錄樹和統計資訊

2. **瀏覽目錄結構**
   - 點擊目錄旁的箭頭圖示可摺疊/展開
   - 查看每個檔案的詳細資訊
   - 目錄以黃色資料夾圖示顯示
   - TXT 檔案以文件圖示顯示

3. **同步到資料庫**（可選）
   - 點擊「同步到資料庫」按鈕
   - 確認同步操作
   - 等待同步完成
   - 查看同步結果

## 🔧 技術細節

### API 端點

#### 1. 掃描目錄
```http
GET /api/comfy-wildcard/scan
```

**回應範例**:
```json
{
  "success": true,
  "data": {
    "name": "Comfy_Wildcard",
    "type": "directory",
    "file_count": 48,
    "total_lines": 1344,
    "children": [...]
  },
  "summary": {
    "total_files": 48,
    "total_lines": 1344,
    "scan_time": "2025-12-07T22:13:30"
  }
}
```

#### 2. 同步到資料庫
```http
POST /api/comfy-wildcard/sync
Content-Type: application/json

{
  "dry_run": false
}
```

**參數**:
- `dry_run`: (可選) 設為 `true` 時只測試不實際匯入

**回應範例**:
```json
{
  "success": true,
  "imported": 1200,
  "skipped": 144,
  "errors": []
}
```

## 📁 Docker 卷映射

`docker-compose.yml` 配置：
```yaml
volumes:
  - E:/Comfy_Wildcard:/app/comfy_wildcard:ro
```

- 將主機的 `E:\Comfy_Wildcard` 映射到容器內的 `/app/comfy_wildcard`
- `:ro` 表示唯讀模式，防止容器修改原始檔案

## 💡 使用場景

### 場景 1: 檢查目錄結構
適用於：想要快速查看 ComfyUI Wildcard 目錄的組織結構

**步驟**:
1. 訪問 ComfyUI 監視頁面
2. 點擊「重新掃描」
3. 瀏覽目錄樹，確認所有分類都正確

### 場景 2: 統計 Wildcard 數量
適用於：想知道每個分類有多少個 wildcard

**步驟**:
1. 掃描目錄
2. 查看統計卡片的總數
3. 展開各個目錄查看具體數量

### 場景 3: 同步新增的 Wildcard
適用於：剛剛在 ComfyUI 目錄中新增了一些 TXT 檔案，想同步到管理系統

**步驟**:
1. 點擊「重新掃描」獲取最新結構
2. 點擊「同步到資料庫」
3. 確認同步操作
4. 查看同步結果（會自動跳過已存在的項目）

### 場景 4: 監控檔案變化
適用於：定期檢查 ComfyUI 目錄的更新

**步驟**:
1. 記錄當前的檔案數和行數
2. 稍後重新掃描
3. 比較統計數據，了解新增了多少內容

## ⚠️ 注意事項

1. **唯讀模式**
   - ComfyUI 目錄在 Docker 中是唯讀掛載
   - 無法通過網頁介面修改或刪除檔案
   - 修改需直接在 `E:\Comfy_Wildcard` 進行

2. **同步行為**
   - 同步時會根據目錄路徑自動分類
   - 使用最後一層目錄名稱作為分類依據
   - 如果分類在資料庫中不存在，wildcard 會被標記為「未分類」

3. **重複檢測**
   - 系統會檢查內容和分類的組合
   - 相同內容在相同分類中只保留一個
   - 跳過的項目會在同步結果中顯示

4. **效能考慮**
   - 掃描大目錄（>100 檔案）可能需要幾秒鐘
   - 同步大量資料可能需要較長時間
   - 建議在低峰時段進行大規模同步

## 🎨 介面特點

### 視覺設計
- **目錄**: 黃色資料夾圖示，藍色左邊框
- **檔案**: 灰色文件圖示，灰色左邊框

- **統計徽章**:
  - 藍色：檔案數量
  - 綠色：行數
  - 灰色：檔案行數

### 互動功能
- **摺疊/展開**: 點擊箭頭圖示
- **懸停效果**: 滑鼠移到項目上會高亮
- **即時更新**: 掃描後立即顯示最新資料

## 🔄 與 ComfyUI Adaptive Prompts 的關係

這個監視功能主要用於：

1. **視覺化管理**: 在一個介面中查看所有 ComfyUI wildcard 檔案
2. **統計分析**: 了解每個分類的 wildcard 數量
3. **資料備份**: 將 ComfyUI wildcard 同步到資料庫作為備份
4. **集中管理**: 在 Wildcard 管理系統中統一管理來自不同來源的 wildcard

**工作流程**:
```
E:\Comfy_Wildcard (ComfyUI 使用)
    ↓ 監視
ComfyUI 監視頁面 (視覺化查看)
    ↓ 同步
Wildcard 管理系統資料庫 (集中管理)
    ↓ 匯出
其他格式 (TXT/JSON/CSV)
```
